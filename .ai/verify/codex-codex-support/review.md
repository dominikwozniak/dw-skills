# Review — Codex support: cross-host hooks, plugin restructure, install/compat validators

Multi-axis review of this change — correctness, readability, architecture, security, performance.
Every finding points at a real `file:line`; a clean axis is "— none —", not an omission. Findings
marked **(reproduced)** were confirmed by executing the code against synthetic inputs; **(plausible)**
means the mechanism is real but the trigger depends on unverified host behavior.

## Verdict

**request-changes** — three high findings, all in the shipped hook/bootstrap layer: the env guard
false-blocks legitimate Codex patches, the local-command extractor turns annotated
`CLAUDE.local.md` command lines into an exit-2 on every edit, and the Codex bootstrap wires hook
scripts it never copies.

## Summary

**request-changes** — the distribution/validation architecture is genuinely solid (version sync,
symlink layout, description budgets, explicit-only parity, and commit conventions all verified
clean), but the shipped hook/bootstrap layer has three blockers, all reproduced: the env guard
false-blocks any Codex patch whose content mentions `.env` (`block-env-access.sh:56`), the
local-command extractor regresses annotated `CLAUDE.local.md` command lines into an exit-2 on every
edit (`hook-common.sh:67`), and the Codex bootstrap wires hook scripts it never copies
(`dw-bootstrap/SKILL.md:96`). Fix the env guard's two findings (fallthrough + `Move to:`) as one
change — the buggy fallthrough is currently the only thing accidentally catching unquoted renames.
The deepest open question is the **plausible** Codex event-shape contract (`codex-hooks.json:5`):
nothing in the repo pins what Codex actually sends, and an argv-array payload demonstrably defeats
both shell guards — worth settling with one captured real event before this ships. Reviewed and
deliberately excluded: the lint-on-edit outside-repo drop is an intentional, security-positive
change (commented in hook-common.sh:37); the double agnix scan in lint.sh is forced by agnix's
single-target CLI; the missing version field in `.agents/plugins/marketplace.json` is a CLAUDE.md
wording nit, not a defect (the Codex marketplace schema has no version slot and the manifest version
is enforced three other ways).
